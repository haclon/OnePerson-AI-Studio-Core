## 🔍 文档概要

### proposal.md - 框架提案

**核心内容**：
- **背景**：为什么需要通用框架
- **问题**：现有方案的局限
- **解决方案**：三层架构（通用层/预设层/实例层）
- **核心机制**：项目自适应注入、审计流程、无损规则适配

**关键决策**：
- 使用 Python 而非 Node.js（更通用）
- 采用模板参数化而非代码生成
- 支持 Init/Inject 两种模式

### analysis-standardization.md - 标准化分析

**核心内容**：
- **参数提取**：从 BOQ 项目中识别可参数化的部分
- **通用层**：诊断模式、模型选择策略、角色框架
- **定制层**：项目名称、技术栈、路径配置
- **10个核心参数**：完整的参数清单

**关键决策**：
- 使用 `{{PLACEHOLDER}}` 作为参数格式
- 预设文件采用 JSON 格式
- 支持多级参数继承

### action-plan.md - 行动计划

**核心内容**：
- **阶段1**：在 BOQ 项目中验证参数化
- **阶段2**：创建独立 GitHub 仓库
- **阶段3**：编写安装脚本和文档
- **阶段4**：在新项目中完整测试

**关键决策**：
- 渐进式开发，每阶段有明确交付物
- 先验证再发布
- 文档优先于代码

## 💡 设计原则（供参考）

从这些文档中提炼的核心设计原则：

1. **简单优于复杂** - 用 JSON + Python 而非复杂框架
2. **参数化优于硬编码** - 一切可配置
3. **文档优先** - 先写清楚，再动手
4. **渐进式增强** - 核心功能先行，高级功能后续
5. **社区友好** - 易于理解、易于贡献
